# Skill: Documentation

## Purpose

Generate and maintain documentation for the multi-tenant-mcp monorepo.

## Capabilities

- **API Documentation** — Generate API docs from TypeScript types and JSDoc comments
- **README Updates** — Update package and root READMEs with feature changes and usage examples
- **CHANGELOG Generation** — Changesets generate CHANGELOGs automatically on version bump
- **Documentation Validation** — Validate documentation consistency and accuracy across packages

## Input Parameters

```json
{
  "action": "generate-api-docs | update-readme | validate-docs",
  "options": {
    "outputDir": "string",
    "package": "string",
    "includePrivate": "boolean",
    "format": "markdown | html"
  }
}
```

## Output

- Generated API docs in `docs/api/`
- Updated READMEs with new content
- CHANGELOGs auto-generated by changesets in each package directory
- Validation report with inconsistencies and suggestions

## Usage Examples

### Update a Package README

```json
{
  "action": "update-readme",
  "options": {
    "package": "rate-limiter",
    "feature": {
      "name": "RedisRateLimitStore",
      "description": "Distributed rate limiting with Redis backend",
      "usage": "import { RedisRateLimitStore } from '@reaatech/multi-tenant-mcp-rate-limiter';"
    }
  }
}
```

### Validate All Documentation

```json
{
  "action": "validate-docs",
  "options": {
    "checkLinks": true,
    "checkExamples": true
  }
}
```

## When to Invoke

- After adding or changing public APIs in any package
- Before release to ensure docs match code
- When README examples become outdated
- When adding a new package to the monorepo

## Invocation Actions

1. Update JSDoc comments in source files
2. Update the relevant `packages/<name>/README.md`
3. Update root `README.md` if the feature spans packages
4. Validate all markdown links
5. Run `pnpm typecheck` to verify type examples compile

## Documentation Standards

### Package README Structure (A2A format)
- Title (`# @reaatech/multi-tenant-mcp-<name>`)
- Three badges: npm version, License, CI status
- Status banner (`> **Status:** Pre-1.0 — ...`)
- One-line description
- Installation (npm + pnpm)
- Feature Overview (bullet list with bold terms)
- Quick Start (runnable TypeScript code example)
- Exports (table: Export | Kind | Description)
- Related Packages (bulleted list of sibling packages)
- License (link to LICENSE)

### Root README Structure (A2A format)
- Title, badges, tagline blockquote
- Overview
- Features table
- Installation (using packages + contributing)
- Quick Start
- Packages table
- Documentation links

### CHANGELOG Format
- Auto-generated by changesets from `.changeset/*.md` entries
- Follows [Keep a Changelog](https://keepachangelog.com/) conventions
- Categorizes: Added, Changed, Fixed, etc.

## Validation Rules

- All public APIs must be documented
- All examples must be syntactically valid
- All links must be functional
- Documentation must match current API
- No outdated version references
- Consistent terminology across all packages

## Configuration

Configured via `skills.config.json`:

```json
{
  "documentation": {
    "apiOutputDir": "./docs/api",
    "includePrivate": false,
    "rootReadmePath": "./README.md"
  }
}
```

## Error Handling

- Generation failure → Error with partial output if any
- Validation failures → Report with specific file/line numbers and suggestions
- Missing documentation → List of undocumented public APIs
- Broken links → List of URLs and source files
